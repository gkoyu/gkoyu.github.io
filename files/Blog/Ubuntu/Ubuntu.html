一、Mac Os 制作Ubuntu的U盘启动盘

由于本人使用MacOS系统，启动盘制作比较麻烦，可以参考其它windows启动盘制作教程比较简单

1. 下载Ubuntu的iso镜像文件

下载地址：https://ubuntu.com/download/desktop

2. 把iso镜像文件转换成dmg文件

cd Downloads/
hdiutil convert -format UDRW -o ubuntu.iso ubuntu-22.04.2-desktop-amd64.iso

format为生成文件的权限；UDRW :表示转换成有read/write的权限的镜像。

mv ubuntu.iso.dmg ubuntu.iso
sudo dd if=./ubuntu.iso of=/dev/disk4 bs=1m

5. 弹出U盘

sudo eject /dev/disk4

二、系统安装与分区方案

三、系统配置与环境配置

1、root密码和sudo设置

1.1 进入系统后设置一下root账户密码

sudo passwd root

1.2 同时为了避免每次sudo都要输入密码，这里配置一下visudo：

sudo visudo

在文件最后加上一句（user_name改为自己的用户名）：

user_name ALL=(ALL) NOPASSWD: ALL

2、配置国内镜像软件源

2.1 搜索tuna tsinghua ubuntu进入

https://mirrors.tuna.tsinghua.edu.cn/help/ubuntu/

找到自己版本Ubuntu的镜像源，这里以Ubuntu22.04为例子

2.2 先备份原来的源：

cp /etc/apt/sources.list /etc/apt/sources.list.bak

2.3 设置源为清华镜像源

sudo gedit /etc/apt/sources.list

内容设置为：

# 默认注释了源码镜像以提高 apt update 速度，如有需要可自行取消注释
deb https://mirrors.tuna.tsinghua.edu.cn/ubuntu/ jammy main restricted universe multiverse
# deb-src https://mirrors.tuna.tsinghua.edu.cn/ubuntu/ jammy main restricted universe multiverse
deb https://mirrors.tuna.tsinghua.edu.cn/ubuntu/ jammy-updates main restricted universe multiverse
# deb-src https://mirrors.tuna.tsinghua.edu.cn/ubuntu/ jammy-updates main restricted universe multiverse
deb https://mirrors.tuna.tsinghua.edu.cn/ubuntu/ jammy-backports main restricted universe multiverse
# deb-src https://mirrors.tuna.tsinghua.edu.cn/ubuntu/ jammy-backports main restricted universe multiverse

# deb https://mirrors.tuna.tsinghua.edu.cn/ubuntu/ jammy-security main restricted universe multiverse
# # deb-src https://mirrors.tuna.tsinghua.edu.cn/ubuntu/ jammy-security main restricted universe multiverse

deb http://security.ubuntu.com/ubuntu/ jammy-security main restricted universe multiverse
# deb-src http://security.ubuntu.com/ubuntu/ jammy-security main restricted universe multiverse

# 预发布软件源，不建议启用
# deb https://mirrors.tuna.tsinghua.edu.cn/ubuntu/ jammy-proposed main restricted universe multiverse
# # deb-src https://mirrors.tuna.tsinghua.edu.cn/ubuntu/ jammy-proposed main restricted universe multiverse

2.4 更新软件列表

sudo apt update
sudo apt upgrade

2.5 安装英伟达显卡驱动

进入系统的图形桌面，打开Software & Updates软件，可以看到标签栏有一个Additional Drivers：

选择第一个安装Nvidia官方驱动（第二个是开源驱动）即可，根据网络情况稍等大概十分钟，安装完重启服务器。

重启完之后更新一下软件：

sudo apt update
sudo apt upgrade

这里会连带Nvidia的驱动一起升级一遍，更新到最新的驱动。

最后输入nvidia-smi命令查看，得到下图，说明显卡驱动安装成功

3、配置SSH&远程桌面

3.1 安装ssh

sudo apt install ssh

3.2 安装xrdp

Xrdp 是一个微软远程桌面协议（RDP）的开源实现，它允许我们通过图形界面控制远程系统。这里使用RDP而不是VNC作为远程桌面，是因为Windows自带的远程桌面连接软件就可以连接很方便，另外RDP在Windows下的体验非常好，包括速度很快（因为压缩方案做得比较好），可以直接在主机和远程桌面之间复制粘贴等等。

sudo apt install xrdp

安装完成xrdp 服务将会自动启动，可以输入下面的命令验证它：

sudo systemctl status xrdp

开启远程连接

四、配置内网穿透

1、首先准备一个带有公网ip的云服务器

我这里使用的是阿里云的轻量应用服务器

2、在云服务器和本地服务器下载frp内网穿透

官方下载地址：https://github.com/fatedier/frp/releases

因为我的Linux服务器为x86架构所以下载的是这个，根据自己实际情况选择

使用wget下载

wget https://github.com/fatedier/frp/releases/download/v0.44.0/frp_0.44.0_linux_amd64.tar.gz

解压

tar -xzvf frp_0.44.0_linux_amd64.tar.gz

2.1 云服务器端配置

进入文件夹

cd frp_0.44.0_linux_amd64

修改frps.ini配置文件

vi frps.ini

编辑配置文件如下，按【i】键编辑，按【Esc】键退出编辑，再按  :wq  退出并保存

[common]
bind_port = 7000    # 云服务器端口号

dashboard_port = 7500   # 监控端口号，可以通过浏览器查看frp的状态以及代理统计信息展示
dashboard_user = xhz   # 监控界面登陆账户名
dashboard_pwd = 136124 # 监控界面登陆密码

不能有注释

[common]
bind_port = 7000    

dashboard_port = 7500   
dashboard_user = xhz   
dashboard_pwd = 136124

设置开机启动并且后台运行

vi /etc/systemd/system/frps.service

修改为如下，注意根据自己路径修改ExecStart

[Unit]
Description=frps daemon
After=syslog.target  network.target
Wants=network.target
[Service]
Type=simple
ExecStart=/root/frp_0.44.0_linux_amd64/frps -c /root/frp_0.44.0_linux_amd64/frps.ini
Restart= always
RestartSec=1min
[Install]
WantedBy=multi-user.target

完成后，可以通过下面的命令进行控制frp服务端

启动frps：sudo systemctl start frps
打开开机自启动：sudo systemctl enable frps
重启应用：sudo systemctl restart frps
停止应用：sudo systemctl stop frps
查看应用的日志：sudo systemctl status frps

运行下面的命令启动并设置开机启动

systemctl start frps && systemctl enable frps

到这里frp云服务器的服务端配置完成

2.2 服务器本地端配置

下载相应版本的frp并进入frp目录

修改frpc.ini配置文件

vi frpc.ini

修改如下，其实就是通过公网ip将本地服务器的22和3389端口映射到公网的2022和23389端口

[common]
server_addr = xxx # 你的云服务器的公网ip
server_port = 7000 # 刚刚配置的frp服务端口

[Fusion-ssh]
type = tcp
local_ip = xxx    # 本地服务器IP
local_port = 22
remote_port = 20022

[Fusion-rdp]
type = tcp
local_ip = xxx  # 本地服务器IP
local_port = 3389
remote_port = 23389

在云服务器防火墙设置开放刚刚使用到的端口

设置开机启动并且后台运行（后面步骤与云服务器相似）

vi /etc/systemd/system/frpc.service

修改为如下，注意根据自己路径修改ExecStart

[Unit]
Description=frps daemon
After=syslog.target  network.target
Wants=network.target
[Service]
Type=simple
ExecStart=/root/frp_0.44.0_linux_amd64/frpc -c /root/frp_0.44.0_linux_amd64/frpc.ini
Restart= always
RestartSec=1min
[Install]
WantedBy=multi-user.target

完成后，可以通过下面的命令进行控制frp服务端

启动frpc：sudo systemctl start frpc
打开开机自启动：sudo systemctl enable frpc
重启应用：sudo systemctl restart frpc
停止应用：sudo systemctl stop frpc
查看应用的日志：sudo systemctl status frpc

运行下面的命令启动并设置开机启动

systemctl start frpc && systemctl enable frpc

到这里frp本地服务器的服务端配置完成

五、安装CUDA和CuDnn

1、安装CUDA

进入nivida官网https://developer.nvidia.com/cuda-toolkit-archive

根据实际情况选择自己的CUDA版本

我这里选择的是11.8

复制命令下载安装

wget https://developer.download.nvidia.com/compute/cuda/11.8.0/local_installers/cuda_11.8.0_520.61.05_linux.run
sudo sh cuda_11.8.0_520.61.05_linux.run

前面安装了nvidia显卡驱动前面两个取消 X代表选中

添加新的cuda到环境变量:

sudo gedit ~/.bashrc

在其打开的文件的末尾添加，根据自己实际的路径修改

export CUDA_HOME=/usr/local/cuda 
export PATH=$PATH:$CUDA_HOME/bin 
export LD_LIBRARY_PATH=/usr/local/cuda-11.8/lib64${LD_LIBRARY_PATH:+:${LD_LIBRARY_PATH}}

保存，退出，刷新环境变量

source ~/.bashrc

验证cuda:

nvcc -V

得到

nvcc: NVIDIA (R) Cuda compiler driver
Copyright (c) 2005-2022 NVIDIA Corporation
Built on Wed_Sep_21_10:33:58_PDT_2022
Cuda compilation tools, release 11.8, V11.8.89
Build cuda_11.8.r11.8/compiler.31833905_0

2、安装cuDNN

进入官网https://developer.nvidia.com/zh-cn/cudnn根据自己情况选择对应版本

解压

tar -xvf cudnn-linux-x86_xxxxxx_cuda11-archive.tar.xz

复制cudnn文件到cuda

sudo cp cudnn-*-archive/include/cudnn*.h /usr/local/cuda/include 
sudo cp -P cudnn-*-archive/lib/libcudnn* /usr/local/cuda/lib64 
sudo chmod a+r /usr/local/cuda/include/cudnn*.h /usr/local/cuda/lib64/libcudnn*

验证安装是否成功，输入

cat /usr/local/cuda/include/cudnn_version.h | grep CUDNN_MAJOR -A 2

得到下面结果

#define CUDNN_MAJOR 8
